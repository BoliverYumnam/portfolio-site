Book Store

— Full-Stack PERN Web App with Data Engineering & Python Web Scraping

This is a complete Full Stack (PERN) web application designed not only as a functional project but also as a guide for beginners who struggle with deploying full-stack apps. The project combines modern web development with practical data engineering, using multiple technologies across the entire stack. The workflow begins with Python-based web scraping, where 50 rows of book data are scraped anonymously from a public website. The cleaned dataset is then inserted into a PostgreSQL table hosted on Supabase, acting as the app’s fully-managed cloud database. The backend is built using Node.js + Express, providing RESTful API routes that serve the scraped data to the frontend. This backend was deployed on Railway, but the Developer plan trial has expired — causing the deployed API to stop functioning even though the front-end and database remain active. (The architecture still works perfectly in development and can be redeployed anytime.) The frontend is built with React and deployed on Netlify, consuming backend routes to display real data stored in Supabase. The app demonstrates full CRUD potential, API integration, React-based data rendering, and end-to-end connectivity between scraping, data storage, backend APIs, and the UI.

Tech Stack & Deployments:
Frontend: React (Netlify)
Backend: Express.js / Node.js (Railway — expired)
Database: Supabase (PostgreSQL)
Data Engineering: Python-driven web scraping pipeline

A practical, real-world demonstration of combining full-stack development, data scraping, and cloud data engineering into a seamless PERN application.

GitHub Repository: https://github.com/BoliverYumnam/fullstackwebapp_withdataengineering
